## 🔴 阻断（5 项 — 会导致开发者写出错误代码）

### 1. 章节编号断裂：§1 → §3，缺少 §2 项目管理

L54 `#### 1. 技能包管理` → L71 直接跳到 `#### 3. 写作工作台`。没有 §2。

当前 L66-69（右键菜单、三级删除、回收站恢复）挂在 §1 技能包管理下，但它们描述的是**项目管理**行为——新建、打开、重命名、删除项目。这是独立功能域，不应是技能包管理的子项。

**修复**：新建 `#### 2. 项目管理`，将 L66-69 移入，补充：新建项目、打开项目、项目列表空状态。

### 2. 状态转移矩阵与阶段③章节目录矛盾

L472-479 定义了统一的状态转移矩阵：所有阶段卡片点击 ⏹ → ⟳（触发 AI 生成）。但 L104 明确说阶段③章节目录点击后："编辑区切换为**章节索引竖列视图**（非 Markdown 编辑器模式）"——不走 AI 生成流程。

**阶段③不遵守通用矩阵。** 开发者如果只看矩阵、不看 L104 的单独描述，会写出错误的阶段③交互。

**修复**：矩阵加一行"阶段③章节目录"专用行，或 L104 显式标注"（阶段③不适用通用状态转移矩阵，行为见 §5）"。

### 3. entry×行为矩阵缺少 AI 自动打开章节文件的条目

L499-503 的入口×行为矩阵定义了 4 种打开方式，但缺少 §5 L111 定义的"点击章节行 → 若章节文件存在且为空 → 自动触发 AI 生成"。矩阵中"从章节索引视图点击"写的是"Markdown 编辑模式 + 触发 AI"，但没有区分"空文件触发 AI"和"有内容仅加载"。

**修复**：矩阵拆分"章节文件为空"和"章节文件有内容"两列。

### 4. 内联操作按钮的触发条件矛盾

L158-160 定义了 AI 消息底部的三个内联操作按钮：`[📝 已写入]`、`[✏️ 手动调整]`、`[🔄 重新生成]`。

但 L157 说"AI 生成内容完成后在对话块底部显示"，L159 说 `[✏️ 手动调整]` "点击 → 切换到编辑模式，编辑区加载对应文件并聚焦"。问题是：**编辑区已经有该文件的标签页打开着怎么办？** 是切换到已有标签页，还是新建一个重复标签页？

**修复**：补充"若该文件已在编辑区有打开标签页 → 切换到该标签页并聚焦；若无 → 新建标签页加载"。

### 5. 窗口关闭行为未定义

L193 写"最小窗口尺寸：800×600"。但以下场景未定义：
- 窗口关闭按钮的行为是什么？隐藏到托盘还是退出应用？
- AI 生成中用户点关闭 → 阻止？提示？静默退出？
- 未保存内容 + 关闭 → 自动保存已定义了（2 秒间隔），但关闭时是否做最后一次强制保存？

**修复**：补充"窗口关闭行为"小节。至少定义：关闭 → 强制保存所有标签页 → 退出应用；AI 生成中关闭 → 提示"AI 正在生成中，关闭将中断生成"。

## 🟠 高风险（7 项 — 开发时会反复返工）

### 6. L66 右键菜单缺少上下文定义

"用户右键项目名 → 弹出菜单"——在哪个页面？这个菜单出现在 Skill 主页（Phase 10）还是工作台（Phase 2-8）？

DEV-PLAN 把 `cQmYV` 右键菜单-项目 分配给 Phase 10。但 Spec 没有明确页面上文。**Phase 8 误把项目右键菜单接入工作台 left-panel 就是因为这个歧义。**

**修复**：L66 改为"在 Skill 主页中，用户右键项目名 → 弹出菜单"。

### 7. L102 总述与 L124 例外自相矛盾

L102（总述）："点击阶段卡片 → 编辑区加载对应文件 + AI 对话区自动发送该阶段预设指令"
L124（例外）："已完成（✅）的阶段卡片点击时，仅加载文件到编辑区，**不触发 AI**"

总述没有提到例外。开发者读到 L102 会形成错误的心理模型——所有卡片都触发 AI——然后在 L124 才被纠正。

**修复**：L102 末尾加"（已完成阶段除外，见 §4 状态标识规则）"。

### 8. chapter_index.md 不存在/为空时无行为定义

L107-114 详细定义了章节解析和容错，但只覆盖了"chapter_index.md 存在且有内容"的情况。未定义：
- chapter_index.md **文件不存在** → 阶段③卡片点击后显示什么？
- chapter_index.md **存在但内容为空** → 竖列视图显示什么？
- chapter_index.md **格式错误**（正则匹配到 0 条）→ 是否提示用户？

**修复**：补充三种异常状态的 UI 行为。

### 9. 文件树中文件重命名的 IPC 时机未定义

L69"文件重命名联动"定义了重命名后的引用更新范围，但未定义：
- 用户在 inline 编辑中按 Enter → 先调 IPC 重命名 → 等 IPC 返回成功 → 再更新 UI？还是乐观更新？
- IPC 重命名失败（权限不足、文件名冲突）→ UI 怎么回退？
- 重命名进行中用户切换到其他标签 → 行为是什么？

**修复**：补充重命名异步流程的时序描述（至少定义成功/失败两条路径）。

### 10. 编辑器右键菜单的"AI 操作"未定义行为边界

L132："🤖 AI 重写选中内容 / ✨ AI 扩写选中段落 / 🔤 AI 润色选中对话"

这三个操作"将选中文本和操作指令一起发送到对话区"。但未定义：
- 发送后 AI 的回复是直接覆盖编辑器中的选中文本，还是仅在对话区展示让用户手动决定？
- L132 后半句说"AI 返回后仅在对话区显示预览，用户手动决定是否应用到文件"——但"应用到文件"的操作是什么？点按钮？还是其他交互？

**修复**：明确定义"预览 → 应用"的交互路径。

### 11. API 配置页面的"返回"路径缺窗口控制按钮行为

L169："API 配置页面为独立全屏视图...顶部包含窗口控制按钮（最小化/最大化/关闭）"

但 Spec 没有说明这个窗口控制是**Electron 窗口的系统按钮**还是**应用内的自定义按钮**。如果是系统按钮，点击关闭会退出整个应用吗？如果是自定义按钮，点关闭是回到工作台还是退出？

**修复**：明确窗口控制按钮的作用域。参考：Linear 的设置页是应用内路由切换，不调用系统窗口控制。

### 12. 双引擎切换时对话区不可见的中间状态

L519-527 定义了跨引擎 Session 桥接的技术方案，但未定义切换过程中的 UI 状态：
- 导出对话历史 + 新引擎初始化 → 这段时间对话区显示什么？加载态？还是保持旧内容不可交互？
- 桥接失败（格式转换异常）→ 显示什么？"AI 上下文丢失，从空白开始"？

**修复**：补充引擎切换的加载态 + 失败态 UI。

## 🟡 中风险（9 项 — 产品完整性问题）

### 13. 缺少"反功能需求"（专业版强制要求）

CLAUDE.md 专业版要求 Spec 包含"不做"列表。当前 Spec 没有。需要明确：哪些功能是**故意不支持**的？以及每项的评估理由。

示例：
| 不做 | 理由 |
|------|------|
| 不支持拖拽章节顺序 | v1 优先 AI 驱动流程，顺序由 chapter_index.md 控制 |
| 不支持批量导出项目 | 单人使用场景，逐项目导出即可 |
| 不支持协同编辑 | 本地优先，离线可用，协同违背核心定位 |

### 14. 缺少性能预算表（专业版强制要求）

L402 提到性能要求但只有两个数字（编辑器加载 < 100ms、AI 流式 < 50ms）。专业版要求 P50/P90/P99 三级目标 + CI 红线。

### 15. 全局空状态/错误状态设计语言缺统一规范

每个功能独立描述了空态（如 L44"空白，提示导入或新建 Skill"、§5 容错），但没有统一的空态/错误态设计规范。不同功能各自决定导致 UI 不一致。

**修复**：加一个"全局交互模式"小节，定义至少：
- 空态：`灰色 text-text-tertiary + 提示文字 + 可选行动按钮` — 所有空态遵循此模板
- 错误态：`暗红背景 + 浅红边框 + 错误描述 + [重试]/[取消] 按钮`
- 加载态：`skeleton / spinner 二选一` — 哪个场景用哪个

### 16. 对话区消息上限未定义

"持久化 Session，上下文保持"——但对话能无限增长吗？1000 条消息后性能如何？是否需要"对话归档"或"上下文窗口截断"？

### 17. 章节列表的滚动行为未定义（63+ 章节场景）

L114 提到"虚拟滚动"，但只在左侧章节列表。§5 的章节目录竖列视图（全页视图，非侧边栏）**没有定义滚动行为**。当章节数达到 50+ 时，是否需要虚拟滚动？是否有快速跳转？

### 18. Token 用量的计费精度未定义

L142 "📊 12.4K tokens"——这个数字是怎么算的？包含 system prompt 吗？包含 tool call 的往返吗？不同模型的 tokenizer 不同，怎么统一计数？

### 19. 设置页面的"自动备份"缺少触发时机定义

L196 "自动备份（每次关闭项目时备份，保留最近 N 份）"——N 是多少？默认值？可配置范围？备份失败通知用户吗？

### 20. Skill 卡片拖拽排序的持久化未定义

L64 "排第一的为默认 Skill"——这个排序持久化到哪？`skill-priority.json`？关闭应用再开，排序还在吗？

### 21. 缺少"无 API Key 时"的全局降级行为

当用户没有配置任何 API 提供商时：点击阶段卡片 → 触发 AI → AI 不可用 → 显示什么？是错误提示，还是引导用户去配置？
