## 使用流程

1. **Phase 编码完成** → Agent 自动运行 CI（tsc + lint + test + E2E + audit）
2. **CI 全部通过** → 从 Spec 提取该 Phase 的交互描述，填充本清单的 A-D 部分
3. **人类 QA 逐条验收** → 对照本清单手动走一遍交互路径
4. **发现 ❌ →** Agent 修复 → CI 重跑 → QA 重验该条
5. **全部 ✅ →** Phase 签收，进入下一 Phase 或发布

---

> **与 Agent 协同**：QA 发现的问题直接告诉 Agent，Agent 走 bug-fixer → code-review → 关闸验证闭环。本清单是"人→Agent"的输入桥接。
